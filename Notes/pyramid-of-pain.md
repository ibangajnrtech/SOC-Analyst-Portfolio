# Hash Values (Trivial)

The hash value is a numeric value of fixed length that uniquely identifies data. A hash value is the result of a hashing algorithm.

## Common hashing algorithms

1) **MD5 (Message Digest, defined by RFC 1321)** - widely used cryptographic hash function with a 128-bit hash value. MD5 hashes are not considered cryptographically secure.
2) **SHA-1 (Secure Hash Algorithm 1, defined by RFC 3174)** - When data is fed to SHA-1 hashing algorithm, SHA-1 takes an input and produces a 160-bit hash value string as a 40 digit hexadecimal number.
3) **The SHA-2 (Secure Hash Algorithm 2)** - The most common SHA-2 variant is arguably the SHA-256 algorithm which returns a hash value of 256-bits as a 64 digit hexadecimal number.

> A hash is not considered to be cryptographically secure if two files have the same hash value or digest.
