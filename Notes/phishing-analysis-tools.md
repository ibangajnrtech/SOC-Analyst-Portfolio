# Identifying Artifacts

Collecting artifacts provides the foundation for a deeper investigation, such as reputation checks, threat intelligence lookups and behavioral analysis.

## Header Artifacts

Take note of the following when analysing email headers:

1) **Sender Email Address**: Where did the email originate from?
2) **Sender IP Address**: What is the source IP, and what does a reverse lookup reveal?
3) **Email Subject Line**: Does it contain urgency or a call to action?
4) **Recipient Email Address**: Who is the intended recipient (TO/CC/BCC)?
5) **Reply-To Email Address**: Where are responses being directed?
6) **Date and time**: When was the email sent?

## Body Analysis

Take note of the following when analysing email bodies:

1) **URLs and hyperlinks**: Identify all links and expand shortened URLs to reveal their true destination
2) **Attachment name(s)**: What files are included, and do their names or extensions appear suspicious
3) **Attachment hash**: Generate a hash value for threat intelligence lookups

# Email Header Analysis
Some of the key information we need to collect can be viewed directly within an email or web client, such as Gmail or Yahoo!.
However, other details, such as the sender’s IP address and Reply-To information, can only be obtained from the email header.

## Mail Header Analysis

[Messageheader](https://toolbox.googleapps.com/apps/messageheader/analyzeheader), part of Google Admin Toolbox, helps analyse email headers. By pasting the full header into the tool, you can quickly extract key details such as the sender's IP address, routing path, and potential misconfigurations.

## IP and URL Reputation Analysis

It is important to determine the origin of any identified IP addresses and assess the reputations of any associated IPs or URLs.

1) [IPinfo](https://ipinfo.io/) is a simple and effective tool for gathering information about an IP address. By entering an IP, you can quickly view details such as its geographic location and associated organisation. This helps analysts determine whether an IP is potentially malicious.

![IPinfo screenshot](https://tryhackme-images.s3.amazonaws.com/user-uploads/616945d482ef350052080da1/room-content/616945d482ef350052080da1-1774336608146.png)

2) [URLScan.io](https://urlscan.io/) is a tool that enables analysts to safely investigate websites without visiting them directly. By submitting a URL, the service simulates a real user browsing session and records all activity generated by the page. It also captures screenshot of the site and provides insight into its behaviour, helping analysts quickly identify suspicious or potentially malicious content.

![URLScan.io screenshot](https://tryhackme-images.s3.amazonaws.com/user-uploads/616945d482ef350052080da1/room-content/616945d482ef350052080da1-1774336606250.png)

3) [Talos IP & Domain Reputation Center](https://talosintelligence.com/reputation_center/) is a threat intelligence tool from Cisco that enables analysts to assess the reputation of IP addresses, domains, and networks. By submitting an indicator, you can quickly assess whether it has been associated with malicious activity and view its classification.

# Email Body Analysis
This is typically where the malicious payload is delivered, either as a hyperlink designed to lure users to a phishing site or as an attachment intended to compromise the system. During analysis, links can be extracted manually from the visible email content or by examining the underlying HTML and raw source to uncover hidden or obfuscated URLs.

## Mail Body Analysis
An effective way to identify URLs in an email is to use URL extraction tools like [convertcsv URL extractor](https://www.convertcsv.com/url-extractor.htm) or [CyberChef](https://gchq.github.io/CyberChef/#recipe=Extract_URLs(false,false,false)) which makes them a versatile option for email analysis.

## Email Attachments
If the email you are investigating contains an attachment, it may be tempting to download it directly using your email client. Attachments should only be downloaded in a controlled environment, such as a virtual machine or sandbox, to reduce the risk of accidental execution. Once safely obtained, you can use tools like the *sha256sum* command in a Linux environment to generate a hash value for further analysis and reputation checks.
Once the hash value has been obtained, several sites can safely assist with analyzing the attached file for example [Talos IP & Domain Reputation Center](https://talosintelligence.com/reputation_center/).
[VirusTotal](https://www.virustotal.com/gui/home/upload) is also a widely used tool that allows analysts to check the reputation of files, URLs, IP addresses and domains using data from dozens of security vendors. By submitting a file or indicator, you can quickly determine whether it has been flagged as malicious and review detailed detection results. VirusTotal also allows users to upload files and submit URLs for analysis, making it a powerful resource for investigating suspicious artifacts.

# Malware Sandboxes

We can use online tools and services known as malware sandboxes, which allow files to be uploaded and analyzed in a controlled environment. This enables us to safely observe the behavior of a potentially malicious file without risking our own systems.
For example, by uploading an attachment from a suspicious email, we can identify the URLs it attempts to contact, any additional payloads it downloads, and other indicators of compromise (IOCs).

## ANYRUN

[ANY.RUN](https://app.any.run/) is an interactive malware sandbox that allows analysts to safely execute and observe suspicious files and URLs in real time.

![ANY.RUN screenshot](https://tryhackme-images.s3.amazonaws.com/user-uploads/616945d482ef350052080da1/room-content/616945d482ef350052080da1-1774337451514.png)

## Hybrid Analysis

[Hybrid Analysis](https://hybrid-analysis.com/) is a free malware analysis sandbox that allows analysts to upload and examine suspicious files in a controlled environment. It provides detailed insights into file behavior, including system changes, network activity, and indicators of compromise, helping analysts better understand the potential impact of a malicious file.

![Hybrid Analysis Screenshot](https://tryhackme-images.s3.amazonaws.com/user-uploads/616945d482ef350052080da1/room-content/616945d482ef350052080da1-1774337450417.png)

## JOESandbox

JOESecurity created the [JOESandbox](https://www.joesandbox.com/#windows) for advanced malware analysis. It performs both static and dynamix analysis on suspicious files and URLs. It generates comprehensive reports that highlight behaviour, indicators of compromise and threat classifications.

# Using PhishTool

[PhishTool](https://www.phishtool.com/) a powerful platform that streamlines phishing investigaitons is desgined for security professionals across multiple disciplines. It automates much of the manual work involved in analyzing suspicious emails. Whether you are a SOC analyst triaging user-reported messages, a threat intelligence analyst collecting indicators, or a researcher investigating phishing kits, PhishTool brings everything together in one place. By combining threat intelligence, OSINT, email metadata, and automated analysis workflows, PhishTool provides a centralized view of phishing activity, helping you quickly identify malicious intent and extract valuable indicators of compromise.

## Identifying Artifacts

Upon uploading an email to PhishTool, you are presented with many of the valuable artifacts that we discussed in the second task. On the right-hand side, the tool provides the rendered HTML view, similar to what you'd see in your inbox, the raw HTML, and also the message source.

By navigating the tabs across the top of PhishTool, you can analyze key components of the email, including authentication results, transmission paths, and any embedded URLs. If the email contains attachments, they can also be reviewed and analyzed directly within the platform, giving you a comprehensive view of the message in one place.
